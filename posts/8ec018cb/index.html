网络流最小割

发表于2020-12-11|更新于2020-12-11|算法笔记

网络流最小割相关

最小割等于最大流，最小割可以用最大流求的，那么重点在于如何建模，将要求的东西转换为最小割。

最小割求方案

随便输出一种方案只要在残量网络上从源点 BFS ，能到的点在 SSS 集合中，不能到的点在 TTT 集合中，∀(u,v),u∈S,v∈T\forall (u,v),u\in S ,v\in T∀(u,v),u∈S,v∈T 且 u,vu,vu,v 满流则该边属于最小割集。

最小割可行边

就是在某个最小割集中出现的边，设边 (u,v)(u,v)(u,v) ，其为可行边的充要条件：

  1. 该边满流 。
  2. 残量网络上不存在 uuu 到 vvv 的增广路。

若不满流显然可以找到另一条更小的满流的限制流量的边。
若满流但在残余网络中能找到入点到出点的路径，那么需要同时割掉这条边和残余网络中路径上某条边才能实现“割”，而这两条边上的流量和一定等于其他某条或某几条满流的边的流量和，否则可以继续增广，又因为位于残余网络上的那条边一定不满流，所以这种割法不是最小割。

判断一条边是否可行直接 BFS 即可，单次 O(n)O(n)O(n)
如果在同一个图上大量判断，可以用 tarjan 在残量网络上求 SCCSCCSCC （强连通分量），若 SCC[u]≠SCC[v]SCC[u]\ne SCC[v]SCC[u]=SCC[v] 且 (u,v)(u,v)(u,v) 满流则说明 该边为可行边。（如果存在 uuu 到 vvv 的增广路，那条路径会和 (u,v)(u,v)(u,v) 反向边构成环，u,vu,vu,v 会在同一强连通分量里）

最小割必须边

就是在任意最小割集中，该边都存在，设边 (u,v)(u,v)(u,v) ，其为可行边的充要条件：

  1. 该边满流
  2. 残量网络上 sss 能到 uuu，vvv 能到 ttt

若不满流显然可以找到另一条更小的满流的限制流量的边。
若满流但在残余网络中源点不能到入点或出点不能到汇点，那么在每条单独路径上一定都存在一条满足最小割可行边的边（容量为 0 阻断路径且没有其他路径），割掉这条可行边是等价的。
必须边的另一种理解是扩大容量后能增大最大流的边。

从 sss BFS，从 ttt 反向BFS，标记能到的点，再枚举满流边即可。
如果求过 SCCSCCSCC ，枚举满流边 判断 SCC[u]=SCC[s],SCC[v]=SCC[t]SCC[u]=SCC[s],SCC[v]=SCC[t]SCC[u]=SCC[s],SCC[v]=SCC[t] 即可（因为有反向边， uuu 到 sss 肯定连通，ttt 到 vvv 肯定连通）

最小割任意方案

每次选取一条满流边，判断其是否为可行边（BFS 一下）
若可行则算入方案，并需要删除它的影响，需要退流 ：uuu 向 SSS 退流， vvv 向 TTT 退流，倒着跑 Dinic 。并删除 (u,v)(u,v)(u,v) 及其反向边，流量都赋零。

求字典序最小最小割 ，见 「SDOI2014」LIS

最少割边最小割

做完一次最小割后，令所有满流边容量为 111，非满流边容量为 ∞\infty∞ 。再做一次最小割，此时的任意最小割方案割边都最少。

最大权闭合子图

闭合图：对于一张有向图 GGG 存在点集 VVV 满足对于任意点 u∈Vu\in Vu∈V 其出边所到点 vvv 都有 v∈Sv\in Sv∈S 。
最大权闭合子图：每个点带上一个权值 valuval_uvalu​ （可正可负），点权和最大的闭合子图就是最大权闭合子图。

建图

  1. 建立超级源汇 s,ts,ts,t ，对于正权点 s→us\to us→u 边权 valuval_uvalu​ ，对于负权点 u→Tu\to Tu→T 边权 ∣valu∣|val_u|∣valu​∣
  2. 对于原图上的边，边权全部为 +∞+\infty+∞

跑最大流，答案等于 ( 正权点之和 −-− 最大流 )

证明：

最小割 === 最大流。 最小割将图划分成两块，于 sss 向连的记 SSS ，剩下的记 TTT 。

SSS 必然是闭合子图，因为原图上的边都是 +∞+\infty+∞ 不可能被割，对于一对入点出点必然同属于 SSS 或 TTT。

对于原图贪心地想，考虑所有正点都选中，然后调整图使其为闭合图 （删尽量少的正点权，加尽量少的负点权），对应到新图上就是删除尽量少的边得到一个闭合子图，就是最小割呗。

答案含义就是： 正权点之和 - (删除最小的正点权和 + 增加的最小的负点权和 )

求方案

考虑割的含义，对于边 s→us\to us→u ，割了这条边表示删了 uuu 这个正点；对于边 v→tv\to tv→t ，割了这条边表示加了 vvv 这个负点。故求完最小割后， SSS 中的点就是最大权闭合子图中的点。

处理集合间的矛盾关系

一个简单的模型：
nnn 个物品，和两个集合 A,BA,BA,B ，每个物品纳入 AAA 或 BBB （二选一），放入 AAA 代价 aia_iai​ ，放入 BBB 代价 bib_ibi​ ，还有若干个限制，描述为 u∈A,v∈Bu\in A,v\in Bu∈A,v∈B 则有 www 的代价。

建图

  1. s→us\to us→u 边权 aua_uau​ ，u→tu\to tu→t 边权 bub_ubu​
  2. 对于限制 u∈A,v∈Bu\in A,v\in Bu∈A,v∈B 有代价 www ： v→uv\to uv→u 边权 www

求最小割即可。
可以理解为，割了 s→us\to us→u 的边 uuu 就属于 AAA ，割 u→tu\to tu→t 的边，uuu 就属于 BBB ，显然不会两条边都割（不然就不是最小割了）。见下图，如果 u∈A,v∈Bu\in A,v\in Bu∈A,v∈B （即割了 边 a,d）此时还需要割 f 才能使 s,ts,ts,t 不连通，故会算上 www 的代价。


上图的建边是一个通用的模型，上图的最小割有四种情况，分别对应 u,vu,vu,v 的属于集合的不同情况：

  1. a,b: u∈A,v∈Au\in A,v\in Au∈A,v∈A
  2. c,d: u∈B,v∈Bu\in B,v\in Bu∈B,v∈B
  3. a,d,f: u∈A,v∈Bu\in A,v\in Bu∈A,v∈B
  4. b,c,e: u∈B,v∈Au\in B,v\in Au∈B,v∈A

然后具体边权就视题目而定了。有些时候不是求最小代价而是求最大贡献，那就反一下，先把所有贡献算上，减去最小的不取的贡献就行了。

喜悦值总和最大 === 未收获喜悦值总和最小

对于某个人 xxx：

  1. 源点向 xxx 连容量为 ( 他选文科的喜悦值 +++ 他和四周的人都选文科的喜悦值/2 ) 的边 。
  2. xxx 向汇点连容量为 ( 他选理科的喜悦值 +++ 他和四周的人都选理科的喜悦值/2 ) 的边 。
  3. xxx 向旁边的所有点连容量为 ( 他们同时选文科 +++ 理科的喜悦值/2 ) 的边 。
